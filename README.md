# Infrastructure réseau sécurisée

## Schéma de l'infrastructure

[![schema](schema/infra.png)](https://github.com/Tontongroudy/Infras-R-seau-S-curis-es/blob/main/schema.png)

L’infrastructure mise en place permet de tester différentes configurations de sécurité réseau et de réaliser des attaques contrôlées afin d’évaluer la robustesse du système.

---

## Organisation du dépôt

Le dépôt est organisé en plusieurs travaux pratiques.

### TP1 – Mise en place de l'environnement

Configuration initiale de l’infrastructure réseau et des machines.

[Tp1](https://github.com/Tontongroudy/Infras-R-seau-S-curis-es/tree/main/tp1)

Contient :
- configuration réseau
- mise en place des services
- tests de connectivité

---

### TP2 – Sécurisation

Mise en place de différents mécanismes de sécurité.

[Tp2](https://github.com/Tontongroudy/Infras-R-seau-S-curis-es/tree/main/tp2)

Exemples :
- règles de filtrage réseau
- configuration sécurisée des services
- restrictions d'accès

---

### TP3 – Tests d’attaque

Tests permettant d’évaluer la sécurité de l’infrastructure.

[Tp3](https://github.com/Tontongroudy/Infras-R-seau-S-curis-es/tree/main/tp3)

Scripts utilisés :
- scan réseau
- identification des services
- tests de vulnérabilités

---

## Objectif

L’objectif de ce projet est de :

- mettre en place une infrastructure réseau sécurisée
- configurer différentes protections
- analyser les résultats d’attaques simulées
- comprendre les vulnérabilités possibles d’un système.

---

## Rendu

Le rendu contient :

- ce README
- les configurations présentes dans les dossiers TP
- les scripts utilisés pour les tests d’attaque.
